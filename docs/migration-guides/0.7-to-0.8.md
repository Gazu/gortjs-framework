# Migration Guide: 0.7.0 to 0.8.0

GortJS `0.8.0` keeps the distributed runtime base from `0.7.0` and adds a stronger developer-facing surface.

## What is new

- official CLI generation flows through `create`, `templates`, `scaffold`, `dashboard`, and `compat`
- browser-based inspector available at `/inspector`
- plugin SDK helpers: `definePlugin`, `createPluginManifest`, and `defineDriverFactory`
- explicit runtime version metadata in `/runtime`
- supported plugin API versions declared more clearly

## Plugin compatibility

`0.8.0` expects plugin API version `0.8`, but still accepts `0.6` plugins for smoother adoption.

Recommended update for plugin modules:

```ts
import { createPluginManifest, definePlugin } from '@gortjs/core';
```

Then set:

```ts
apiVersion: '0.8'
```

## Runtime operations

No change is required for the distributed runtime basics introduced in `0.7.0`:

- cluster roles
- remote command routing
- Redis and webhook runtime event adapters
- persistence adapters `file`, `memory`, and `redis`

## Recommended migration checklist

1. Update package versions to `0.8.0`.
2. Move plugin examples to `apiVersion: '0.8'`.
3. Add the inspector to your operational workflow.
4. Replace ad-hoc starter projects with CLI templates and scaffolds.
