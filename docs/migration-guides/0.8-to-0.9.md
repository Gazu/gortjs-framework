# Migration Guide: 0.8.0 to 0.9.0

GortJS `0.9.0` keeps the adoption improvements from `0.8.0` and adds the operational hardening layer that prepares the framework for `1.0.0`.

## What is new

- structured runtime logs through `GET /logs`
- audit trail through `GET /audit`
- `GET /health/live` and `GET /health/ready`
- request and correlation IDs echoed by REST and propagated into event history
- plugin lifecycle hooks:
  - `start`
  - `stop`
  - `dispose`
  - `healthCheck`
- runtime plugin summaries now include operational state and health

## Plugin API

`0.9.0` expects plugin API version `0.9`, while still accepting `0.8` and `0.6` plugins for migration continuity.

Recommended manifest update:

```ts
apiVersion: '0.9'
```

Optional new hooks:

```ts
start(context) {}
stop(context) {}
dispose(context) {}
healthCheck(context) {
  return { ok: true };
}
```

## Runtime logging

You can now configure runtime logging under:

```json
{
  "runtime": {
    "logging": {
      "enabled": true,
      "console": true,
      "file": "./runtime/runtime.log",
      "auditFile": "./runtime/audit.jsonl",
      "maxEntries": 200
    }
  }
}
```

## Recommended migration checklist

1. Update package versions to `0.9.0`.
2. Move plugin manifests to `apiVersion: '0.9'` where possible.
3. Add runtime logging configuration for production-oriented deployments.
4. Point orchestration probes to `/health/live` and `/health/ready`.
5. Use `x-request-id` and `x-correlation-id` when integrating external clients.
